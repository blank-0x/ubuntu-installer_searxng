Automated deployment script for SearXNG using Docker on Ubuntu systems.

## Overview

This project provides a reproducible and controlled installation method for SearXNG. It avoids common issues found in upstream installers by using containerization and adding validation, logging, and configuration flags.

The script handles:

- System validation (Ubuntu + root check)
- Docker installation with duplicate detection
- Port validation (range 1–65535)
- Automatic `SECRET_KEY` generation and injection
- SearXNG container deployment via Docker Compose
- Post-deploy health check
- Structured logging with fallback path

---

## Requirements

- Ubuntu 20.04 / 22.04 / 24.04
- `sudo` / root access
- Internet access
- `curl` and `openssl` available

---

## Installation

### Clone the repository

```bash
git clone https://github.com/blank-0x/ubuntu-installer_searxng.git
cd ubuntu-installer_searxng
```

### Make the script executable

```bash
chmod +x install_searxng.sh
```

---

## Usage

### Default install (port 8080, installs Docker)

```bash
sudo ./install_searxng.sh
```

### Custom port

```bash
sudo ./install_searxng.sh --port 9090
```

### Skip Docker installation (if already installed)

```bash
sudo ./install_searxng.sh --no-docker-install
```

### Custom port without reinstalling Docker

```bash
sudo ./install_searxng.sh --port 9090 --no-docker-install
```

### Help

```bash
sudo ./install_searxng.sh --help
```

---

## Post-installation

If you were added to the `docker` group during installation, apply permissions:

```bash
newgrp docker
```

Or log out and back in.

## Service Management

```bash
# Start
docker compose -C /opt/searxng up -d

# Stop
docker compose -C /opt/searxng down

# Status
docker compose -C /opt/searxng ps

# Logs
docker compose -C /opt/searxng logs -f
```

## Configuration

| File | Purpose |
|------|---------|
| `/opt/searxng/.env` | Environment variables and `SECRET_KEY` |
| `/opt/searxng/docker-compose.yml` | Container configuration |
| `/opt/searxng/core-config/` | SearXNG settings directory |

## Logs

```
/var/log/searxng-installer.log
```

> Falls back to `/tmp/searxng-installer.log` if `/var/log` is not writable.

---

## Rationale

The official SearXNG installer does not support all distributions and may fail in environments such as Kali Linux due to:

- Distribution-specific assumptions
- Python packaging restrictions (PEP 668)
- Missing service integrations

This project uses Docker to provide a consistent and isolated runtime environment.

---

## Production Considerations

For non-local deployments:

- Set a secure `SECRET_KEY` (auto-generated by default)
- Use a reverse proxy such as NGINX or Caddy with TLS
- Restrict access via firewall rules or HTTP authentication
- Consider rate limiting and monitoring
